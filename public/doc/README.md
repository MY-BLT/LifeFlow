# LifeFlow – دستیار هوشمند تحلیل بهره‌وری

**نام پروژه:** LifeFlow  
**توسعه‌دهنده:** محمدیاسین بریدلقمانی طوسی  
**شماره دانشجویی:** ۴۰۲۲۲۳۷۳  

---

### 📝 شرح کوتاه ایده
LifeFlow یک پلتفرم تحلیل بهره‌وری مبتنی بر هوش مصنوعی است که برخلاف ابزارهای سنتی (مانند Todoist یا تسکولو)، به کاربران کمک می‌کند تا **علت ریشه‌ای (Root Cause)** شکست در دستیابی به اهداف را کشف کنند. این سیستم با تحلیل الگوهای رفتاری (خواب، شبکه اجتماعی، تمرکز و ساعت کاری)، فرسودگی شغلی را پیش‌بینی کرده و بینش‌های عملی برای بهبود عملکرد فردی و سازمانی ارائه می‌دهد. تمام پردازش‌ها به صورت **محلی (Offline-first)** انجام می‌شود تا حریم خصوصی کاربران به طور کامل حفظ شود.

---

### 🛠 فناوری‌های استفاده‌شده
- **فریم‌ورک اصلی:** React 18 + TypeScript
- **ابزار ساخت (Build Tool):** Vite
- **استایل‌دهی:** Tailwind CSS (به همراه Custom CSS Variables در فایل `index.css`)
- **فونت:** Vazirmatn (بهینه‌شده برای زبان فارسی)
- **ذخیره‌سازی داده:** LocalStorage (بدون نیاز به Backend)
- **مدیریت حالت:** React Hooks (useState, useEffect)
- **هوش مصنوعی:** Rule-based AI Engine (تشخیص الگو و تولید بینش)

---

### 🚀 نحوه اجرای پروژه
برای اجرای پروژه روی سیستم محلی، مراحل زیر را دنبال کنید:

1.  **کلون یا استخراج** فایل‌های پروژه در یک پوشه.
2.  نصب وابستگی‌ها:
    ```bash
    npm install
    ```
3.  اجرای سرور توسعه:
    ```bash
    npm run dev
    ```
4.  باز کردن مرورگر و رفتن به آدرس نمایش‌داده‌شده (معمولاً `http://localhost:5173`).

> **توجه:** برای ساخت نسخه نهایی (Build) از دستور `npm run build` استفاده کنید.

---

### 🧠 بخش‌های پیاده‌سازی‌شده توسط دانشجو
- **معماری کلی و روترینگ:** طراحی و پیاده‌سازی `App.tsx` و سیستم ناوبری.
- **موتور تحلیل (`analytics.ts`):** محاسبه امتیازات ۵‌گانه (بهره‌وری، تمرکز، سلامت، تعادل و کنترل اتلاف) و تولید بینش‌های هوشمند.
- **تمامی صفحات تحلیلی:** طراحی کامل صفحات `IdeaPage`، `MaturityPage`، `ValuePage`، `BusinessModelPage`، `MarketPage`، `RevenuePage`، `MarketingPage` و `RoadmapPage` با محتوای تخصصی کسب‌وکار.
- **سیستم ذخیره‌سازی (`storage.ts`):** پیاده‌سازی عملیات CRUD برای فعالیت‌ها، اهداف و عادت‌ها.
- **داشبورد سازمانی (`OrgDemoPage`):** طراحی پنل مدیریت تیم، پروژه‌ها، گلوگاه‌ها و گزارش اجرایی.
- **داشبورد شخصی (`PersonalDemoPage`):** پیاده‌سازی تقویم شمسی، نمودارهای پیشرفت، سیستم اهداف و عادت‌ها.
- **سیستم طراحی (`index.css`):** نوشتن کلاس‌های سفارشی، انیمیشن‌ها، دارک مود و ریسپانسیو.

---

### 🤖 بخش‌های تولیدشده با کمک هوش مصنوعی (AI-Assisted)
با استفاده از هوش مصنوعی (مدل‌های زبانی بزرگ) سرعت توسعه افزایش یافت و کیفیت کد بهبود داده شد:

- **کد پایه کامپوننت‌های سنگین:** ساختار اولیه صفحات `PersonalDemoPage` و `OrgDemoPage` (شامل گریدها، کارت‌ها و منطق تب‌ها) با کمک AI تولید و سپس توسط دانشجو شخصی‌سازی و بهینه‌سازی شد.
- **دیباگ و رفع خطاهای تایپ‌اسکریپت:** رفع خطاهای تایپ (Type Errors) در بخش‌های پیچیده مانند `generateInsights` و مدیریت `useState` در کامپوننت‌های بزرگ.
- **محتوای متنی صفحات تحلیلی:** بخش‌هایی از محتوای توضیحی (مانند تحلیل رقبا و آمارهای بازار) با استفاده از AI نگارش و سپس با توجه به نقشه‌های ذهنی دانشجو ویرایش و بومی‌سازی شد.
- **ابزارهای کمکی:** تولید کدهای تکراری مانند کامپوننت‌های `ScoreRing`، `MiniSparkline` و `HeatmapChart` با رویکرد "Prompt to Code" انجام شد.

---

### ⚠️ محدودیت‌های نسخه فعلی (MVP)
- **عدم وجود Backend واقعی:** تمام داده‌ها صرفاً در مرورگر (LocalStorage) ذخیره می‌شوند. با پاک کردن حافظه مرورگر، داده‌ها از بین می‌روند.
- **هوش مصنوعی مبتنی بر قانون (Rule-based):** موتور AI فعلی بر اساس الگوهای از پیش تعریف‌شده کار می‌کند و از مدل‌های یادگیری عمیق واقعی (Deep Learning) استفاده نمی‌کند.
- **نیاز به ورود دستی داده:** کاربران باید فعالیت‌های خود را به صورت دستی ثبت کنند (اتوماسیون کامل در نسخه‌های آینده برنامه‌ریزی شده است).
- **عدم یکپارچگی با ابزارهای خارجی:** در نسخه فعلی امکان اتصال به تقویم گوگل، Slack یا Jira وجود ندارد.
- **بدون کاربر واقعی (تست‌شده با داده‌های Sample):** این نسخه صرفاً برای نمایش قابلیت‌ها و اثبات مفهوم (Proof of Concept) طراحی شده است.

#### ۳. توضیحات مربوط به دارایی‌های گرافیکی (Assets)
- **لوگو:** فایل `lifeflowlogo.png` در پوشه `public/` قرار دارد. در صورت عدم دسترسی به فایل اصلی، می‌توانید از یک آیکون ساده با پس‌زمینه گرادینت (طبق کد موجود در `Navbar`) استفاده کنید.
- **فونت:** فونت `Vazirmatn` به صورت `@import` در فایل `index.css` یا از طریق CDN لود می‌شود.
- **آیکون‌ها:** تمام آیکون‌های استفاده‌شده در پروژه (مانند ایموجی‌های 📊، 🧠، ⚡) از کتابخانه استاندارد یونیکد (سیستم‌عامل) استفاده می‌کنند و نیاز به فایل جداگانه ندارند.

---

#### ۴. توضیح نحوه استفاده از هوش مصنوعی
در فرآیند توسعه این پروژه، از هوش مصنوعی به عنوان **دستیار برنامه‌نویس (Coding Assistant)** در کنار دانش فنی دانشجو استفاده شده است:

1.  **تولید کد پایه (Boilerplate):** برای شروع سریع‌تر پروژه، ساختار اولیه صفحات واکن‌گرا (Responsive) مانند گرید کارت‌های صفحات `MarketPage` و `CompetitorPage` توسط AI پیشنهاد و سپس با اصول طراحی UI/UX دانشجو تطبیق داده شد.
2.  **رفع خطاهای منطقی:** در بخش‌هایی مانند حلقه‌های تودرتو برای محاسبه `groupByCategory` یا رندر شرطی در جداول، از AI برای یافتن راه‌حل بهینه استفاده شد.
3.  **بهبود مستندات:** محتوای فایل `README.md` و بخش‌های توضیحی داخل کد (Comment) با کمک AI نگارش و استانداردسازی شد.
4.  **تولید داده‌های نمونه (Sample Data):** داده‌های ساختگی برای اعضای تیم، فعالیت‌ها و پروژه‌ها در فایل `sampleData.ts` با استفاده از پرامپت‌های تخصصی تولید گردید تا تنوع داده‌ها برای نمایش در دموها حفظ شود.

**تأکید:** تمامی کدهای تولیدی توسط AI، **سطر به سطر** توسط دانشجو بازبینی، ویرایش و بهینه‌سازی شده‌اند تا با معماری کلی پروژه و نیازمندی‌های کسب‌وکار هماهنگ باشند و دانشجو تسلط کامل بر کلیت کد دارد.
